## ğŸ“ Technical Architecture

### Frontend Stack
- **Framework**: Next.js 15 (React 19)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **State**: Zustand + React Query
- **Forms**: React Hook Form + Zod
- **Date**: date-fns + tz
- **Rich Text**: Tiptap (Markdown)
- **Calendar**: FullCalendar or react-big-calendar
- **Kanban**: @dnd-kit/core
- **Charts**: Recharts
- **Icons**: Lucide React

### Backend Stack
- **API**: Next.js API Routes (tRPC for type safety)
- **Database**: PostgreSQL (Prisma ORM)
- **Auth**: NextAuth.js
- **File Storage**: S3-compatible (Cloudflare R2 or Wasabi)
- **Real-time**: Pusher or LiveKit
- **Email**: Resend
- **Queue**: BullMQ (Redis)
- **Cache**: Redis

### Mobile Apps
- **Framework**: React Native (with Expo)
- **Navigation**: Expo Router
- **State**: Zustand
- **Sync**: Custom sync engine

### Desktop Apps
- **Framework**: Electron or Tauri
- **Windows**: Native installer
- **macOS**: DMG with auto-update
- **Linux**: AppImage, deb, rpm

### Infrastructure
- **Hosting**: Vercel (frontend) + Railway/Fly.io (backend)
- **Database**: Managed PostgreSQL (Neon or Supabase)
- **CDN**: Cloudflare
- **Monitoring**: Sentry (errors) + Vercel Analytics
- **CI/CD**: GitHub Actions
